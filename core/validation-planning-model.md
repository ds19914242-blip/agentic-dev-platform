# Validation Planning Model

## Definition

Validation Planning defines how work will be tested and verified.

## Purpose

Implementation must be validated against requirements before completion.

## Validation Plan

A Validation Plan may include:

- acceptance criteria
- test strategy
- manual checks
- automated tests
- security checks
- performance checks
- regression checks

## Flow

Specification
→ Architecture Proposal
→ Task Breakdown
→ Validation Plan
→ QA Execution

## Rules

Validation Plan must exist before final implementation review.

High-risk work requires stronger validation.

Failed validation blocks completion.
