# Evaluation Model

## Definition

Evaluation is the process of determining whether execution results meet expectations.

## Purpose

The platform must verify quality before progressing to the next stage.

## Evaluation Targets

The system may evaluate:

- Specifications
- Architecture Proposals
- Task Breakdowns
- Code Changes
- Test Results
- Deployments
- Agent Outputs
- Execution Graph Results

## Evaluation Sources

Evaluation may use:

- acceptance criteria
- tests
- review results
- security checks
- performance checks
- policy compliance
- human feedback

## Evaluation Results

- passed
- passed_with_warnings
- failed
- requires_review

## Evaluation Fields

Each evaluation should define:

- target
- evaluator
- criteria
- result
- findings
- recommendations
- timestamp

## Rules

Important artifacts must be evaluated before approval.

Failed evaluations block progress.

Evaluation results must be recorded.

High-risk work requires stronger evaluation.
