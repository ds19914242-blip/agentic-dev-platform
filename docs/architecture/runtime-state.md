# Runtime State

The platform repository should contain source code, product configs, docs, and stable templates.

Runtime-generated state should not be treated as source code.

## Runtime directories

These directories are generated by platform execution:

- backlog/
- runs/
- evidence/
- tmp/

They can become very large and should not be committed by default.

## Target direction

Long term target:

agentic-dev-platform/
  source code only

../agentic-dev-data/
  backlog/
  runs/
  evidence/

## Why

Keeping runtime state outside the repository makes the platform easier to maintain:

- cleaner git status
- smaller commits
- easier code review
- less accidental artifact tracking
- clearer separation between control-plane and runtime state
