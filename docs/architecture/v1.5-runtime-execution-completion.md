# v1.5 Runtime Execution Completion

## Added

- runtime feature path can execute writes with --execute-writes
- validation runs after implementation in the runtime graph
- acceptance runs before review so review can evaluate acceptance evidence
- recovery foundation can retry implementation/validation once with --recovery
- LLM planner option exists through --llm-planner
- legacy agents remain archived and unused by active command path

## Safety

Default runtime execution is dry-run.

Real code writing requires:

--execute-writes

Real validation/acceptance requires:

--execute
